Tutoriales de Git Flow

Aprende a colaborar en proyectos de software usando Git.

Objetivos del Proyecto

  • Comprender el flujo de trabajo de Git Flow.
  • Aprender a crear y gestionar ramas en Git.
  • Colaborar y realizar revisiones de código usando pull requests.

Contenido

Este repositorio contiene tutoriales sobre:

  • Creación y uso de ramas (`feature`, `release`, `hotfix`)
  • Buenas prácticas para los mensajes de commit
  • Cómo hacer pull requests y revisiones de código

Cómo Empezar

Para comenzar a utilizar este proyecto:

  1. Clona el repositorio: git clone https://github.com/tu-usuario/tutoriales-git-flow.git
  2. Cambia a la rama principal: git checkout main
  3. Sigue las instrucciones de cada tutorial.

Es posible que, al ser la dueña del repositorio original, GitHub no permita hacer un fork de tu propio repositorio. En este caso, se puede trabajar directamente en el repositorio principal sin necesidad de hacer el fork y simular el flujo de trabajo de git flow creando y trabajando en ramas para cada tarea.

Crea una Rama Nueva para la Primera Tarea: Abre la terminal y asegúrate de estar en el directorio de tu repositorio principal. Crea una nueva rama para actualizar el README.md

Actualiza el Archivo README.md:

  • Abre README.md en un editor de texto y agrega más detalles sobre el proyecto.
  • Guarda los cambios.
  • Haz Commit de los Cambios en la Nueva Rama:
  • luego Sube la Rama a GitHub:

    • Crear una Pull Request (PR) en GitHub Ve a tu repositorio en GitHub y verás una notificación que te sugiere crear una pull request desde la rama feature/actualizar-readme hacia main. Haz clic en “Compare & pull request”. Completa la descripción de la PR con detalles de los cambios que hiciste. Crea la pull request.
